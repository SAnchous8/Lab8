# Задание 1

### Текст задачи
Каталог книг

Разработать консольное приложение с дружественным интерфейсом с возможностью выбора заданий для работы с «базой данных (БД)», хранящейся в бинарном файле. Перечень полей (минимум 5), достаточно полно характеризующих заданную в варианте предметную область, предложить самостоятельно (постараться отразить в перечне полей такие, которые требуют разных типов данных). Приложение должно выполнять следующие функции:
1. Чтение базы данных из бинарного файла.
2. Просмотр базы данных.
3. Удаление элементов (по ключу).
4. Добавление элементов.
5. Реализация 4 запросов (формулировки запросов придумать самостоятельно). 2 запроса должны возвращать перечень, 2 запроса одно значение. В классе должны присутствовать свойства, конструкторы, перегруженный метод ToString(). Весь функционал приложения реализовать в виде методов вспомогательного класса с помощью LINQ запросов.

### Алгоритм решения

1. Запуск приложения  
   Программа начинает выполнение с метода Main(). Выводится название приложения «Каталог книг».

2. Определение пути к файлу базы данных  
   Формируется путь к бинарному файлу books.dat в папке с программой.

3. Загрузка данных из файла  
   Вызывается метод BookDatabase.LoadFromFile(filePath).  
   - Если файла нет — создаётся пустой список книг.  
   - Если файл есть — из него последовательно читаются все записи (количество записей, затем для каждой книги — ID, название, автор, год, цена, страницы, жанр).  

4. Проверка наличия данных  
   - Если список пуст, пользователю предлагается заполнить базу вручную (меню выбора).  
   - Если не пуст — выводится количество загруженных книг.

5. Главный цикл программы  
   Программа переходит в бесконечный цикл while (running), в котором отображается главное меню.

6. Главное меню  
   Пункты меню:  
   1 — Просмотр всех книг  
   2 — Добавить книгу  
   3 — Удалить книгу (по ID)  
   4 — Запросы  
   5 — Сохранить базу в файл  
   6 — Загрузить базу из файла  
   7 — Заполнить базу вручную  
   0 — Выход  

7. Пункт 1. Просмотр всех книг  
   Вызывается метод PrintAll(books), который выводит все книги в табличном виде с разделителями и подсчётом общего количества.

8. Пункт 2. Добавление книги  
   - Запрашиваются поля: ID, название, автор, год, цена, страницы, жанр.  
   - Проверяется уникальность ID.  
   - Создаётся новый объект Book.  
   - Вызывается метод AddBook(books, newBook), который добавляет книгу в список.  

9. Пункт 3. Удаление книги  
   - Запрашивается ID книги для удаления.  
   - Вызывается метод RemoveBook(books, id), который ищет книгу по ID и удаляет её.  

10. Пункт 4. Меню запросов  
    Отображается подменю с четырьмя запросами:  
    1 — Книги заданного жанра (перечень)  
    2 — Книги дороже заданной цены, отсортированные по убыванию цены (перечень)  
    3 — Средняя цена всех книг (одно значение)  
    4 — Общее количество страниц всех книг (одно значение)  

    Запрос 1. Книги по жанру  
    - Пользователь вводит жанр.  
    - Вызывается метод GetBooksByGenre(books, genre).  
    - Внутри метода используется Where(b => b.Genre.ToLower() == genre.ToLower()).  
    - Результат выводится в виде перечня.

    Запрос 2. Книги дороже цены  
    - Пользователь вводит минимальную цену.  
    - Вызывается метод GetExpensiveBooks(books, minPrice).  
    - Внутри метода используется Where(b => b.Price > minPrice).OrderByDescending(b => b.Price).  
    - Результат выводится в виде перечня.

    Запрос 3. Средняя цена  
    - Вызывается метод GetAveragePrice(books).  
    - Внутри метода используется Average(b => b.Price).  
    - Выводится одно значение (средняя цена).

    Запрос 4. Общее количество страниц  
    - Вызывается метод GetTotalPages(books).  
    - Внутри метода используется Sum(b => b.Pages).  
    - Выводится одно значение (сумма страниц).

11. Пункт 5. Сохранение в файл  
    Вызывается метод SaveToFile(filePath, books), который записывает список книг в бинарный файл.

12. Пункт 6. Загрузка из файла  
    Вызывается метод LoadFromFile(filePath), который перезагружает данные из файла.

13. Пункт 7. Ручное заполнение базы  
    - Очищается текущий список книг.  
    - Запрашивается количество книг для ввода.  
    - Для каждой книги последовательно вводятся все поля с проверкой уникальности ID.  
    - После ввода всех книг данные сохраняются в файл.

14. Пункт 0. Выход из программы  
    - Запрашивается подтверждение на сохранение изменений.  
    - Если пользователь вводит «да», вызывается SaveToFile(...).  
    - Цикл running завершается, программа закрывается.

15. Вспомогательные методы безопасности ввода  
    - ReadInt() — безопасный ввод целого числа с обработкой пустого ввода и ошибок формата.  
    - ReadDouble() — ввод вещественного числа с автоматической заменой точки на запятую.  
    - ReadString() — ввод строки с проверкой на пустое значение.

16. Обработка ошибок  
    - При чтении или записи файла перехватываются исключения Exception, выводится сообщение об ошибке.  
    - При вводе с клавиатуры обрабатываются пустой ввод и неверный формат.  
    - При создании книги через конструктор проверки на корректность данных не выполняются, но могут быть добавлены при необходимости.  

17. Конец работы программы
